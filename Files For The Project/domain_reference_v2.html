Workspace Aggregate Root
Изолированное пространство команды — корень всей системы
📋 Полей: 14🏷️ Enum'ов: 1🔗 Связей: 4⚡ CQRS: 3C/2Q
▶

Workspace — главный Aggregate Root всей платформы. Изолированная «песочница» компании или агентства. Все данные системы — аккаунты соцсетей, видео, посты, автоматизации, диалоги — принадлежат Workspace через прямую или транзитивную связь по WorkspaceId.

Реализует multi-tenancy: данные разных клиентов хранятся в одной БД, но изолированы. Каждый API-запрос проходит middleware, который добавляет WorkspaceId из JWT ко всем EF Core запросам.

Plan управляет тарифными лимитами. При превышении лимитов бизнес-логика сервисов возвращает ошибку ещё до записи в БД.

02
ApplicationUser Aggregate Root
Учётная запись человека — может быть в нескольких Workspace с разными ролями
📋 Полей: 14🔗 Связей: 3⚡ CQRS: 4C/2Q
▶

ApplicationUser — учётная запись конкретного человека. Существует независимо от Workspace. Один человек — Owner в своей компании и Editor в клиентской — связь через WorkspaceMember.

Не наследует от IdentityUser — избегаем зависимости Domain Layer от инфраструктуры. Хэширование пароля через IPasswordHasher в IAuthService.

TimeZoneId (IANA) только для отображения дат в UI. Все даты в БД — UTC. Soft delete через IsActive=false: история постов и сообщений сохраняется.

WorkspaceMember Entity
Роль пользователя в Workspace — основа RBAC
📋 Полей: 12🏷️ Enum'ов: 2🔗 Связей: 3⚡ CQRS: 4C/2Q
▶

WorkspaceMember — junction entity между User и Workspace с бизнес-логикой RBAC. Owner (полный доступ + billing), Admin (команда + аккаунты), Editor (посты), Viewer (только чтение).

Процесс приглашения: InviteMemberAsync → WorkspaceMember{Invited} → email с HMAC-токеном → AcceptInviteAsync → Active. Инвариант: Owner не может понизить свою роль или удалить себя.

RefreshToken Entity
JWT Refresh Token сессии — SHA-256 хэш, ротация, управление устройствами
📋 Полей: 10🔗 Связей: 1⚡ CQRS: 4C/1Q
▶

RefreshToken — запись об активной пользовательской сессии. Стратегия ротации: при использовании IsUsed=true + создаётся новый. Повторное использование IsUsed=true токена = детекция кражи → инвалидируются ВСЕ токены пользователя.

В БД хранится только SHA-256 хэш. Plaintext токен существует только в памяти и передаётся клиенту в HttpOnly Cookie — даже утечка БД бесполезна злоумышленнику.

SocialAccount Aggregate Root
Подключённый аккаунт соцсети — мост ко всем внешним платформам
📋 Полей: 23🏷️ Enum'ов: 2🔗 Связей: 5⚡ CQRS: 4C/2Q
▶

SocialAccount — мост между системой и внешними платформами (YouTube, Instagram, TikTok, Facebook, Twitter, Telegram). Три потока: публикация видео, получение сообщений в Inbox, автоматизации. Один Workspace может иметь несколько аккаунтов одной платформы.

OAuth-токены шифруются через IDataProtectionProvider. TokenService.EnsureTokenValidAsync() прозрачно обновляет через refresh_token при истечении — до вызова Publisher'а.

IsPrivateAccount критично для PendingDMQueue: Instagram запрещает DM приватным аккаунтам без взаимной подписки.
